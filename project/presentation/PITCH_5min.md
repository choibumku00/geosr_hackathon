# 발표 5분 초안 — `validate-model-output` (수치모델 후처리·검증 자동화)

- A팀 / 발표자: (전원 분담, 아래 §분담)
- 형식: 슬라이드 7장 + 라이브 데모. 총 ~5분(±). 각 장에 **[발표멘트]** 와 **[타이밍]**, **(채점항목)** 표기.
- 채점 나침반: 과정70(①활용25 **②재현·전파25** ③전원10 ④문제10) + 결과30(⑤완성15 ⑥임팩트12 ⑦자유3). 동점 시 ②→①→③.
- 데모 실패 대비: **사전 녹화**를 `submit/evidence/`에 준비(발표 전 운영자 고지), 라이브는 **오프라인 소형 fixture**로.

---

## S0. 타이틀 (10s)
- **수치모델 후처리·검증, 매번 손으로 → 한 번에 일관되게**
- A팀 · 공통과제: "모델 출력 후처리·검증 자동화"
- [발표멘트] "예보 업무에서 하루에도 여러 번 반복하는 '모델 출력 검증'을, 누가 언제 돌려도 똑같이 나오는 스킬로 만들었습니다."

## S1. 문제 (35s) (④문제정의)
- 수치/AI 모델 출력을 **후처리·검증**하는 게 일상 반복 업무.
- 포맷·규약이 제각각: NetCDF3/NetCDF4·HDF5/CSV, 좌표 1D/2D/비정형mesh, 단위 K/°C, 영문/한글.
- 지금은 CDO·NCO·Python·ncview·wgrib2를 **손으로 오가며 눈으로** → 느리고, **세션마다 절차가 달라지고**, 통과/실패 **근거가 안 남는다**.
- [발표멘트] "핵심 문제는 '느림'이 아니라 **'매번 달라짐'** 입니다. 같은 파일도 그때그때 다른 도구·다른 기준으로 보니, 재현도 전파도 안 됩니다."

## S2. 핵심 통찰 — 맨손 Claude와 무엇이 다른가 (45s) (①활용깊이 ②재현성)
- 그냥 에이전트한테 시키면? → **매번 코드·패키지·임계가 바뀜**(껍데기 위험).
- 우리 해법 = **역할 분업**:
  - 흔들리면 안 되는 검증 규칙 → **결정적 Python + yaml에 박제**(완전 재현)
  - 판단(어느 게 모델/기준, 어떤 분석, 결과 해석) → **에이전트 유도 대화**
- 분석의 다양성 = **검증 방법 카탈로그(방법 ~500개 · 그림 ~115개)** 를 메뉴로.
- [발표멘트] "재현돼야 하는 곳은 코드로 박제하고, 판단이 필요한 곳만 에이전트가 합니다. 이게 맨손 Claude가 못 하는 **일관성**입니다 — 대회 최고 배점이 재현·전파성인 이유와 정확히 맞습니다."

## S3. 어떻게 동작하나 (40s) (①활용깊이)
- **4페이즈**: `발견(discover) → 질문유도(elicit) → 다축검증(verify) → 리포트(report)`. **분석을 즉시 시작하지 않고** 무엇이 있는지부터 파악·되묻는다.
- **직교 yaml**: `rules.yaml`(QC 임계) ⟂ `recipes.yaml`(도메인별 분석) ⟂ `aliases.yaml`(변수·단위·한글 별칭).
- **canonical metrics 단일구현** + **§G 함정 강제**(기준자료≠참값·advisory 임계·단일지표 금지).
- [발표멘트] "포맷·도메인을 자동 판별하고, 한 번 짠 지표를 전 도메인이 재사용합니다. 새 자료는 yaml만 고치면 됩니다."

## S4. 라이브 데모 ★ (110s) (⑤완성도 ⑥임팩트 ③전원)
> 오프라인 fixture로 실행. (M3·M4 완료 시 전체, 미완 시 QC+discover 중심 + 녹화 보조)

**데모 ①-discover (20s):** `discover project/data/`
→ ERA5·GFS(기상)·WW3(파랑 mesh)·부이(CSV)를 **포맷·도메인·역할 자동 분류**.
[멘트] "사용자가 설명 안 해도, 무엇이 모델이고 무엇이 기준인지 먼저 파악합니다."

**데모 ②-QC 결함적발 (35s):** `validate 정상본` → **PASS 12/12** / `validate 결함본` → **FAIL**: 기온 500K·풍속 999·격자 비단조를 **근거와 함께 적발**, 손상파일은 **무크래시**.
[멘트] "정상은 통과, 일부러 망가뜨린 파일은 조목조목 적발 — 검증을 '진짜' 합니다." (실측 검증 완료)

**데모 ③-파랑 다축 verify (45s):** `verify WW3 --ref 부이`
→ 부이 지점에 모델 **최근접 매칭** → Hs **bias·RMSE·SI**·QQ·**파향 원형통계**·시계열, **2024-09-15 고파 이벤트** 피크 비교, 해역별 표.
[멘트] "모델과 실관측을 같은 지점·시각으로 짝지어 다축으로 검증하고, 리포트엔 '부이=점관측·기준자료≠참값' 경고가 자동으로 붙습니다."

**데모 ④-재현 (10s):** 같은 명령 재실행 → **동일 리포트**.
[멘트] "몇 번을 돌려도 같은 결과 — 이게 일관성입니다."

## S5. 재현·전파 (40s) (②재현·전파성 — 최고배점)
- 스킬은 **자기완결 폴더** → 복사만으로 옆 사람 PC에서 동작.
- 새 변수·도메인·기준자료 = **yaml만 수정(코드 0줄)**.
- **부서 자산**: 검증 방법 카탈로그(15파일) + 그림 카탈로그(7도메인) — 따라 할 레시피.
- [발표멘트] "방법론을 한 사람 머릿속이 아니라 **재사용 자산**으로 남겼습니다. 이게 전파입니다."

## S6. 효과 — Before / After (30s) (⑥임팩트)
| 지표 | Before(맨손) | After(스킬) |
|---|---|---|
| 같은 요청 반복 | 매번 다름 | **매번 동일**(재현율 0%→100%) |
| 검증 항목 | 누락 가능 | **항상 전 항목** |
| 결함 적발 | 눈으로 | **자동+근거** |
| 전파/확장 | 머릿속 | **폴더복사·yaml** |
- [발표멘트] "효과를 시간이 아니라 **일관성·재현성**으로 측정했습니다 — 우리 업무의 진짜 문제였으니까요."

## S7. 한계(정직) + 마무리 (20s) (④신뢰성)
- 해석 임계는 **advisory**(영역·해상도 의존), 기준자료는 **reference(≠참값)**, 부이는 대표성오차.
- [마무리 멘트] "완성된 AI가 아니라, **반복 수작업을 일관·재현 가능하게 바꾼** 도구입니다. 누가 언제 돌려도 같은 검증 — 그게 우리 답입니다."

## 리스크·대비
- **사전 녹화**(데모③ 우선) `submit/evidence/`에 — 발표 전 운영자 고지.
- 라이브는 **오프라인 소형 fixture**(대용량 원본 의존 금지).
- M3·M4 미완 시: 데모①②(완성·검증됨) 중심 + 데모③는 녹화/프로토타입 결과로 대체, "다축 설계·카탈로그는 완비" 강조.

## 슬라이드 제작 메모
- 장당 글자 최소, **데모 화면이 주인공**. 코드보다 **리포트(PASS/FAIL·근거)** 와 **그림(Hs SI·파향 rose)** 을 크게.
- 색: 결함=빨강 FAIL, 정상=초록 PASS 대비. 마지막 장에 "재현율 0%→100%" 한 줄.
