# 일관성 미니실험 결과 — 맨손 즉석검증 vs 스킬

> 목적: '재현율 Before≈0% / After=100%'를 실측으로 보인다. (발표 S2·S6 뒷받침)

## PART A — 즉석(ad-hoc) 검증의 변동성 (같은 결함 파일, 다른 구현 → 다른 결론)

결함 파일 `era5_demo_broken.nc` (주입: t2m 9칸=500K, NaN블록, u10=999, 격자 비단조).
아래는 '검증해줘'에 대해 사람마다 그럴듯하게 짤 법한 4가지 즉석 구현이다.

| # | 즉석 구현(그럴듯한 선택) | 결론 |
|---|---|---|
| 1 | 평균 기온 상식범위? | 평균 t2m=300.3K → PASS  (500K 9칸이 평균에 안 묻혀 **놓침**) |
| 2 | 최댓값<330K? | 최대 t2m=500K → FAIL  (값범위로 잡음) |
| 3 | 결측 있으면 FAIL? | np.mean=nan(NaN전파)·결측존재=True → FAIL  (결측은 잡지만 500K·격자는 안 봄) |
| 4 | 격자 단조성만? | 위도 단조=False → FAIL  (격자는 잡지만 500K·결측은 안 봄) |

**→ 같은 파일인데 결론이 PASS/FAIL로 갈림**: ['PASS', 'FAIL', 'FAIL', 'FAIL']  (PASS 1 · FAIL 3)
→ 어떤 항목을 점검하느냐(구현 선택)에 따라 **결함을 통째로 놓칠 수 있다.** 이게 '맨손'의 비일관성.

### 덤: 같은 정상파일의 '평균 2m기온'도 방법 선택에 따라 다른 숫자
| 방법 | 값 |
|---|---|
| 단순 평균(면적 미가중) | 300.159 K |
| cos(위도) 면적가중 평균 | 300.248 K |
| 첫 시각만 평균 | 299.652 K |
| °C로 환산(부주의 시 단위혼동) | 27.009 °C |
→ 다 '평균 기온'이지만 **정의·선택이 다르면 다른 값**. 보고서마다 숫자가 달라지는 이유.

## PART B — 스킬 `validate`의 결정성 (같은 입력 → 바이트까지 동일)

`validate era5_demo_broken.nc` 3회 실행 → report.md SHA-256:

- 실행 1: `9118b44616b285a0fe73bf73…`
- 실행 2: `9118b44616b285a0fe73bf73…`
- 실행 3: `9118b44616b285a0fe73bf73…`

**→ 3회 해시 동일 = True** (재현율 100%). 같은 입력이면 리포트가 바이트까지 같다.

## 결론
- **Before(맨손)**: 같은 결함파일 → 즉석 구현 4개가 1:3로 **불일치**, 결함 놓침 발생.
- **After(스킬)**: 같은 입력 3회 → 리포트 **완전 동일**(True). 검증 항목 고정·근거 자동.
- 효과지표 = **일관성·재현성**: 재현율 Before≈0% → After=100%.
