# validate-model-output

**수치모델·AI·위성 등 어떤 결과물이든, 그 데이터에 맞는 검증 방식을 스스로 찾아 다각도로 검증하는 범용 스킬.**

특정 분야(파랑·해류 등)에 최적화되지 않고, **지구시스템 전반**(대기·해양·파랑·해수면·해빙·수문·대기질·위성 EO … + 이를 만드는 AI)을 아우르도록 설계했다. 격자 데이터든 정점(관측소)·시계열 데이터든, 심지어 카탈로그에 아직 없는 분야든 동작한다.

---

## 이 스킬이 하는 일

수치·AI·위성 결과물을 검증할 때 매번 반복되는 문제 —
- 파일 열기·변수·단위·시간대를 손으로 확인하고, 비교·플롯을 매번 새로 코딩
- 분야·포맷·규약이 제각각(K/°C, 1D/2D/mesh, 한글 cp949, KST vs UTC)
- 검증자마다 지표·임계가 달라 **비일관·재현 불가**, 그리고 단순 RMSE에서 멈춤

— 을, 에이전트가 데이터를 파악·적응하고 **논문 기반 다축 검증**을 반복 심화하며 재현 가능한 보고서로 만들어 해결한다.

---

## 우리가 구현한 것 (핵심 설계)

1. **데이터 특성 기반 범용 라우팅** — 분야를 "목록에서 고르는" 방식이 아니라, **자료형**(격자/mesh/정점·시계열/프로파일/스펙트럼)과 **변수성질**(벡터·원형·양수/꼬리·분율·앙상블)에서 검증법이 따라 나온다. 그래서 카탈로그에 없는 분야도 동작한다. (`characterize()`)

2. **미지 도메인 안전망 — 절대 멈추지 않음** — 분야를 못 정해도 틀린 분야를 끼워 맞추지 않고, 데이터 특성으로 **일반 검증 배터리 + 트리거**를 구성해 진행한 뒤 사용자에게 확인한다.

3. **능동 메타탐색 + 되묻기** — 사용자가 놓치기 쉬운 것(시간대 KST/UTC, 각 컬럼 의미, 변수 단위, 정점 좌표 출처)을 에이전트가 먼저 탐색하고, 모르는 것은 되물어 확정한 뒤에만 분석한다.

4. **논문 근거 카탈로그 내장** — `references/research/`에 분야·그림별 검증 방법 카탈로그(약 500 방법카드·115 그림카드, 출처·DOI 포함). 단순 RMSE가 아니라 **정말 세부적인** 검증을 문헌 근거로 수행한다.

5. **반복 심화 루프** — 1차 검증에서 끝내지 않는다. 결과를 판독해 약점(꼬리·위상·공간·계절 편향)을 찾고, 그에 맞는 2차 검증을 추천·수행하며 **품질을 계속 끌어올린다** (PHASE 4 DEEPEN).

6. **cross-domain 추천** — 파일의 분야에서 관행이 아니어도 그 데이터에 유용한 타 분야·공통 검증법(극치·스펙트럼·Taylor·확률검증 등)까지 함께 추천한다.

7. **격자 + 정점·시계열 동급 1급** — 검조소·부이·관측소 CSV 같은 정점 시계열도 격자와 동등하게: 정점별+통합 지표, 시계열/위상, 분포, **정점 위치 지도**.

8. **지도 그림엔 위치정보 필수** — 위경도 좌표 그림(정점 위치·bias 맵·트랙)은 **해안선·육지 + 위경도 라벨**을 반드시 넣어 "어디인지" 식별되게 한다. (오프라인이어도 격자선으로 안전 동작)

9. **AI·위성 = 얹는 축** — AI·ML·위성을 별도 분야가 아니라 **어느 물리 도메인 위에나 얹히는 평가축**(UQ/OOD·매치업/대표성오차 등)으로 다룬다.

10. **샘플코드 적응 원칙** — 내장 코드는 완성품이 아니라 **적응의 출발점(SAMPLE)**. 사용자 데이터 구조가 다르면(변수명·좌표·단위·시간규약) 전처리·메타확인·검증 전 단계에서 그 자리에 맞춤 코드를 작성해 돌린다.

11. **전파성(확장 용이)** — 새 분야는 `config/domains.yaml`에 블록 하나만 추가하면 라우터가 자동 인식한다. (가이드·템플릿: `references/extending.md`)

12. **해석 함정(§G) 강제** — 모든 보고서에 "기준자료≠참값 / 임계는 advisory / 단일 지표 금지 / 동화·보간 산물 독립관측 취급 금지 / 대표성 오차"를 자동 명시한다.

## 흐름 (5단계)

```
DISCOVER → ELICIT → ANALYZE → DEEPEN(반복) → REPORT
 파악        질문      다축검증    2차 심화       보고
```

- **DISCOVER** — 포맷·구조·좌표·역할을 파악(자동 분석 금지). 미지 구조는 그 자리에서 점검.
- **ELICIT** — 역할·도메인·기준자료·시간대·좌표·산출물 종류를 확인(확정 전 분석 금지).
- **ANALYZE** — 데이터 특성으로 다축 배터리 구성(정확도·분포·시간·방향·공간·종합) + cross-domain.
- **DEEPEN** — 결과를 판독해 다음 검증을 추천·수행, 수렴까지 반복.
- **REPORT** — JSON + Markdown 보고서, §G 함정 캡션 강제, 지도엔 해안선+위경도.

## 구성

- `SKILL.md` — 에이전트 진입점(위 흐름·규칙·데이터특성→방법 규칙).
- `references/` — `research/`(논문 카탈로그·그림카드) · `recipe_*`(도메인 레시피) · `plotting_maps.md`(지도) · `extending.md`(새 분야 추가) · `adapting.md`·`architecture.md`.
- `config/` — `domains.yaml`(분야 판별) · `aliases.yaml`(한글·비표준 변수명) · `rules.yaml`(QC 규칙). 모두 YAML만 편집해 확장.
