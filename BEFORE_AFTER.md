# Before / After — 수작업 → 에이전트화

## 우리가 자동화한 반복 수작업
모델 후처리 및 검증 자동화: 수치모델 결과를 ERA5/GLORYS, 관측소 CSV, 위성자료와 비교하기 위한 전처리·통계·시각화·리포트 작성 절차를 범용 Skill로 재사용 가능하게 정리/자동화한다.

## Before (기존 수작업 절차)
1. 검증 대상 파일마다 담당자가 즉석으로 스크립트를 짜거나, 기존 스크립트를 복붙해서 수정한다.
2. 점검 항목(값 범위 / 결측 / 격자 단조성 등)을 그때그때 선택하기 때문에 구현마다 다른 항목을 본다.
3. 같은 결함 파일도 구현 선택에 따라 결론이 달라진다 — 실측: 동일 파일 4구현 → PASS/FAIL 1:3 불일치, 결함 통째 놓침.
4. "평균 기온" 하나도 단순평균/면적가중/첫시각/단위혼동 선택에 따라 300.159 K / 300.248 K / 299.652 K / 27.009 °C 로 갈린다 — 보고서마다 숫자가 다름.
5. 다룰 수 있는 포맷(NetCDF3/4 한글경로, CSV cp949, 비정형 mesh)과 도메인(기상/해양/파랑)을 확인하는 과정이 각자 다르고 문서화가 없다.

## After (에이전트화된 절차)
1. `validate` 서브커맨드 하나로 NetCDF3/4(한글경로 포함)·CSV(cp949/euc-kr/latin-1 폴백)·비정형 mesh 파일을 자동 포맷 감지하여 읽는다.
2. 도메인(기상·해양 격자·파랑 mesh·부이 CSV)을 router가 자동 판별하고 해당 recipe의 QC 규칙을 고정 적용한다.
3. 검증 항목(값범위·결측·격자단조·위경도·시간축 등)이 rules.yaml로 고정되어 누가 실행해도 같은 항목을 점검한다.
4. 실측: 동일 파일 3회 실행 → report.md SHA-256 완전 동일 (재현율 100%).
5. 근거(QC 판정·통계·도메인·타임스탬프)가 report.md에 자동 기록된다. 수작업 캡처 불필요.
6. `verify` 서브커맨드로 모델 vs 기준자료(재분석·관측·위성)를 **다축 비교**한다: 단위 정규화(°C↔K)·격자 정렬·(점관측이면 비정형 mesh→부이 지점 최근접 매치업)·시간대 정규화 후 정확도(bias/RMSE/SI/r)·분포(QQ)·시간·방향(원형통계)·종합(Taylor)·해역(crop)을 함께 산출한다(**단일 지표 금지, 최소 3축 + §G 함정 자동 경고**).
7. 결과물 자체가 **재사용 자산**이다 — Claude Code 스킬로 등록하면 에이전트가 "이 모델 출력 검증해줘"에 스스로 호출하고, 새 변수·도메인은 `config/*.yaml`만 추가해 확장한다(코드 무수정).

## 효과 (재현성 미니실험 실측 결과)

> 근거: `project/scenario/consistency_experiment/RESULT.md` — 동일 결함파일(`era5_demo_broken.nc`) 대상 즉석 4구현 vs 스킬 `validate` 3회 실행 비교.

### 실측 데이터

**PART A — 즉석구현 4개, 같은 결함파일, 다른 결론**

| 구현 # | 즉석 선택 | 결론 | 놓친 결함 |
|--------|----------|------|----------|
| 1 | 평균 기온 상식범위? | **PASS** (500K 9칸이 평균에 희석 → 놓침) | 값이상·결측·격자 |
| 2 | 최댓값 < 330K? | FAIL | 결측·격자 |
| 3 | 결측 있으면 FAIL? | FAIL | 500K·격자 |
| 4 | 격자 단조성만? | FAIL | 500K·결측 |

→ **같은 파일인데 PASS/FAIL 1:3 불일치. 구현 #1은 결함을 통째로 놓쳤다.**

**PART B — 스킬 validate 3회, SHA-256 동일**

```
실행 1: 9118b44616b285a0fe73bf73…
실행 2: 9118b44616b285a0fe73bf73…
실행 3: 9118b44616b285a0fe73bf73…
→ 3회 해시 동일 = True (재현율 100%)
```

> 효과 서술: 검증 항목이 구현자·시점에 따라 달라지는 '맨손' 방식에서, 항목·정의·단위가 고정된 스킬로 전환함으로써 **일관성과 재현성이 Before≈0%에서 After=100%로 개선**되었다. 이는 동일 결함을 놓쳤던 임의성을 제거하고, 보고서마다 숫자가 달라지는 문제를 없앤다.
